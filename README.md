# Hop Category Demo

Демо автоматического определения категорий для объявлений двуязычной доски объявлений (русский + узбекский). Принимает текст объявления и через LLM определяет, к какой категории сайта оно относится.

Проект сделан как демонстрация подхода: показать качество классификации на реальных объявлениях ещё до полноценной интеграции.

## Что делает

- принимает текст объявления (заголовок и описание);
- определяет одну категорию из списка категорий сайта;
- работает и на русском, и на узбекском языке;
- возвращает категорию и уверенность модели;
- сверяет ответ модели с настоящей категорией и считает точность;
- генерирует наглядный HTML-отчёт с таблицей и сырыми ответами модели.

## Как работает

```
текст объявления
      │
      ▼
промпт со списком категорий
      │
      ▼
LLM (gpt-4o-mini)
      │
      ▼
JSON: {"category": "...", "confidence": 0.95}
      │
      ▼
сравнение с эталонной категорией → точность → HTML-отчёт
```

Эталонная категория берётся из структуры ссылки объявления на сайте, поэтому есть с чем сравнивать ответ модели.

## Стек

- Python
- OpenAI-совместимый API (модель `gpt-4o-mini`)
- `python-dotenv` для хранения ключа

## Установка

```bash
git clone https://github.com/burlaevrodion13-dot/hop-category-demo.git
cd hop-category-demo

python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

pip install openai python-dotenv
```

## Настройка ключа

В корне проекта создайте файл `.env`:

```
LLM_API_KEY=ваш_ключ
```

Базовый адрес API задаётся в коде (`base_url`). По умолчанию используется OpenAI-совместимый прокси.

Файл `.env` добавлен в `.gitignore` и не попадает в репозиторий.

## Запуск

1. Впишите тексты объявлений в список `ADS` внутри `demo.py` (поле `text`).
2. Запустите:

```bash
python demo.py
```

3. В консоли появится таблица с результатами и точностью, а в папке проекта файл `demo_result.html` с оформленным отчётом.

Проверить только доступ к модели, без полного демо:

```bash
python test.py
```

## Структура

```
demo.py            # основная логика: классификация и отчёт
test.py            # проверка доступа к API
demo_result.html   # сгенерированный отчёт (создаётся после запуска)

.gitignore
README.md
```

## Результаты демо

На выборке из 12 реальных объявлений (русские и узбекские) категория определена верно в 10 случаях, без предварительной донастройки под полный список категорий.

- Двуязычность работает: одно и то же объявление на русском и узбекском классифицируется одинаково.
- Спорные случаи (например, товар для туризма против услуги-тура) решаются простыми правилами под конкретные категории сайта.

## Ограничения

- Это демонстрация подхода, а не готовая продакшн-система.
- Выборка маленькая, точность здесь иллюстративная, а не строгая метрика.
- Используются категории верхнего уровня.
- Запросы к LLM платные, расходы на стороне владельца API-ключа.

## Безопасность

Ключ API хранится только в `.env` и никогда не коммитится. Если ключ случайно попал в репозиторий, его нужно отозвать в личном кабинете провайдера и создать новый.
